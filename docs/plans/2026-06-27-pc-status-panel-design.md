# PC Status Panel 設計書

作成日: 2026-06-27
ステータス: 設計確定（実装前）

## 1. 概要

AIDA64 の SensorPanel 風に、PC のハードウェアステータスをサブモニタへ
フルスクリーンで常駐表示するアプリ。表示するだけでなく、**アプリ内で自由に
レイアウト・項目・デザインを編集できるエディター**を備える。

- 表示形態: サブモニタ用フルスクリーンダッシュボード
- 性質: **プレイヤー兼エディター**（自分専用の SensorPanel Manager + ビューア）
- 技術: Tauri (Rust) + Svelte/TypeScript + Konva.js、センサーは
  LibreHardwareMonitor をサイドカー同梱

## 2. アーキテクチャ（3層）

```
Tauri アプリ (フルスクリーン / サブモニタ常駐)
├── Frontend (Svelte + TS + Konva.js)
│   ├── キャンバス（編集モード ⇔ 表示モード を切替）
│   ├── レンダラー（ゲージ/グラフ/バー/ラベル/画像）
│   └── 値更新ストア
│        ▲ invoke / event (IPC)
│        ▼
├── Rust Core (Tauri backend)
│   ├── .sensorpanel パーサー（インポート専用）
│   ├── Panel JSON（自前ネイティブ形式）の読込/保存
│   ├── センサー集約・AIDA64センサーID変換
│   └── サイドカー起動・監視・再起動
│        ▲ stdout(JSON) / 名前付きパイプ
│        ▼
└── LHM Sidecar (.NET, LibreHardwareMonitorLib)
    └── ハードウェア列挙 → 1秒間隔で全センサー値を JSON 出力
       （一部センサーは管理者権限が必要。権限はこのプロセスに限定）
```

データフロー: Sidecar(1秒) → Rust(マッピング適用) → Frontend(イベント受信→差分再描画)。
グラフ履歴は Rust 側でリングバッファ保持。

## 3. データモデル

アプリ本体のデータは**独自 JSON 形式**。`.sensorpanel` はインポート時に
この形式へ変換する（ネイティブ形式ではない）。

```rust
enum ItemKind { Label, SensorText, Gauge, GraphLine, BarH, BarV, Image }

struct Rect { x: f32, y: f32, w: f32, h: f32 }

struct Style {
    font_family: String,   // 各アイテムごとに独立（グローバル設定にしない）
    font_size: f32,
    font_weight: String,   // normal / bold
    color: String,         // 例 "#00ffcc"
    align: String,
}

// 画像ベースゲージの方式（針回転 / スプライト切替）
enum GaugeRender {
    NeedleRotate {              // ダイヤル画像 + 針画像を値で回転
        dial_asset: Option<String>,   // 背景ダイヤルPNG（任意）
        needle_asset: String,         // 針PNG（必須）
        pivot: (f32, f32),            // 針の回転中心（針画像内の相対座標）
        angle_start: f32,             // min時の角度（度）
        angle_end: f32,               // max時の角度（度）
    },
    SpriteStrip {               // コマ画像の帯を値で切替
        sprite_asset: String,         // 帯PNG
        frame_count: u32,
        orientation: String,          // horizontal / vertical
    },
    VectorArc,                  // 画像未割当時のデフォルト描画（Konva描画）
}

struct PanelItem {
    id: String,
    kind: ItemKind,
    rect: Rect,                  // 絶対座標
    z: i32,                      // 重なり順
    opacity: f32,                // 0.0〜1.0（半透明オーバーレイ用）
    sensor_src: Option<String>,  // AIDA64センサーID 互換キー（例 "TCPU"）
    style: Style,
    range: Option<(f64, f64)>,   // ゲージ/バーの min/max
    asset: Option<String>,       // Image種別/単一画像参照（アセット名）
    gauge: Option<GaugeRender>,  // kind==Gauge のときの描画方式
    format: Option<String>,      // 表示フォーマット（例 "%d °C"）
}

struct Panel {
    size: Rect,                  // パネル全体の解像度
    background: Option<String>,  // 背景画像（アセット名）
    items: Vec<PanelItem>,
}
```

### アセットライブラリ（素材プール）
画像は「1個のスキンが全体をロックする」のではなく、**各アイテムが個別に参照**する。
これにより、ゲージはスキンA・背景はスキンB・装飾はスキンC、と自由に混在できる。

- 取り込み: 任意の場所の PNG/JPG をアプリに取り込むと
  `~/PCStatusPanels/<panel>/assets/` にコピーして管理（元ファイル移動に非依存）
- 参照: アイテムは `asset` / `gauge.*_asset` でアセット名を参照。複数アイテムで共有可
- 一覧 UI: 取り込み済み素材をサムネイル一覧で表示し、ドラッグで割り当て
- **透過**: アルファ付き PNG をそのまま透過合成。`opacity` で半透明も可。z順で多層重ね合わせ
  （背景 → 透過装飾 → ダイヤル → 針 → テキスト、と何層でも）

### フォントの独立性（要件）
`Style.font_*` は **PanelItem ごとに独立**。テキストボックスAは Orbitron 24px 緑、
ボックスBは Meiryo 16px 白、のように1つずつ自由に設定できる。
プロパティパネルでは「選択中アイテムのフォントだけ」を変更する。
全体フォントのような縛りは設けない。

## 4. エディター UX

```
┌──────┬────────────────────────────┬──────────┐
│ パレット│        キャンバス            │ プロパティ │
│ ▢ ラベル│  選択中=8方向ハンドル        │ 種別/座標 │
│ ◔ ゲージ│  ドラッグ移動・リサイズ       │ センサー割当│
│ ∿ グラフ │  グリッドスナップ            │ min/max  │
│ ▭ バー  │  ▶表示 でフルスクリーンへ     │ 色/フォント│
│ 🖼 画像 │                            │          │
└──────┴────────────────────────────┴──────────┘
```

- 左パレットからキャンバスへドラッグして追加
- クリック選択 → ハンドルでリサイズ、ドラッグで移動
- グリッドスナップ、矢印キー微調整、複数選択、コピペ、z順前後
- 右プロパティで色・フォント・min/max・表示フォーマット・センサー割当を編集
- **編集中も実センサー値が動く**（針が動く・グラフが流れる）
- `▶表示` でフルスクリーン表示モード（ハンドルを消してロック）
- **キャンバスは1つ**。編集モードと表示モードを切り替えるだけ（プレイヤーは
  編集モードのロック版）

### 状態管理
- Undo/Redo: 編集操作を単一責任のコマンドオブジェクトとして履歴に積む
- 唯一の真実は `Panel` JSON
- 保存先: `~/PCStatusPanels/*.json`。複数パネル保持、起動時に最後のパネルを表示

## 5. 描画技術・フロントエンド

- **描画方式: Canvas（Konva.js）** — 選択トランスフォーマ・ドラッグ・当たり判定・
  レイヤー・z順が標準装備でエディターと好相性。ゲージ円弧/針・折れ線・バー・
  画像合成すべて Canvas で描画。Konva はフレームワーク非依存。
- **フレームワーク: Svelte + TypeScript** — バンドル小・常駐向き・状態管理が素直。
  （機能的制約は Svelte/React で差なし。軽さ優先で Svelte を採用）
- **更新と描画の分離**: センサー値更新（1秒）と描画アニメ（requestAnimationFrame）を
  分離。値はストアに入れ、Konva レイヤーは差分のみ再描画（全消し再描画はしない）。
- **2つのストア**:
  - ストアA = `Panel` JSON（構造・編集対象）
  - ストアB = センサー値の最新スナップショット（高頻度更新）
  - 描画層は A（構造）と B（値）を購読

## 6. センサー取得

- **LibreHardwareMonitor をサイドカー同梱**。LHMLib を使う小さな .NET コンソールが
  ハードウェアを列挙し、1秒間隔で全センサー値を JSON で stdout 出力。
- 温度・電圧・ファン回転数まで広く取得可能。一部センサーは管理者権限が必要。
- センサーピッカー: LHM が拾った全センサーを一覧から項目に割り当てる。
- AIDA64 センサーID（例 SCPUUTI, TCPU）→ LHM センサーへのマッピング表を Rust が保持。

## 7. エラー処理（根本対処）

- **Sidecar 落ち検知**: Rust が検知し指数バックオフで自動再起動。再起動中は
  「センサー未接続」状態を通知し値を「--」表示（黙って固まらせない）。
- **管理者権限不足**: 取れた範囲で動かしつつ「権限不足で温度/電圧が未取得」と明示。
- **センサー欠落**: 割当先が存在しないアイテムは「⚠ センサー未検出」と表示。
- **不正/破損 JSON**: 握りつぶさずエラー内容を表示。空パネルで上書きフォールバック
  しない（破損ファイルを消さない）。
- **未対応アイテム種別**: 黙って捨てず、本来の位置に「⚠ 未対応: <種別>」の
  プレースホルダ枠を描画（実装漏れを画面上で可視化）。

## 8. テスト方針（振る舞い重視）

- Rust: パーサー（JSON⇔Panel）、センサーIDマッピング、リングバッファのユニットテスト
- フロント: コマンド（Undo/Redo）、座標スナップ計算、値フォーマットのユニットテスト
- センサー値はモック注入可能にし、実機なしでもテスト可能に

## 9. 進め方（エディター最優先）

- **Phase 1**: キャンバス＋アイテム追加/選択/移動/リサイズ＋プロパティ（フォント独立）
  ＋保存読込。ゲージは VectorArc デフォルト描画、センサーは**ダミー値**で先行。
- **Phase 1.5**: アセットライブラリ（PNG取り込み・透過合成・opacity）＋
  画像ゲージ（針回転・スプライト切替）＋背景画像。
- **Phase 2**: LHM サイドカー＋実センサー配線＋センサーピッカー。編集中に実値が動く。
- **Phase 3**: フルスクリーン表示モード＋常駐（トレイ・自動起動・サブモニタ配置記憶）。
- **Phase 4**: `.sensorpanel` インポート。実ファイルで形式を確定。
